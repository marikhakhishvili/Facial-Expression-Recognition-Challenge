# Facial Expression Recognition (FER2013) Challenge

ეს პროექტი შესრულებულია ნეირონული ქსელების დავალების ფარგლებში PyTorch-ის გამოყენებით. ამოცანის მიზანია სახის გამომეტყველების ამოცნობა (7 კლასი) **FER2013** დატასეტზე. ექსპერიმენტების მენეჯმენტისა და ვიზუალიზაციისთვის გამოყენებულია **Weights & Biases (W&B)** პლატფორმა.

---

## 🏗 რეპოზიტორიის სტრუქტურა

პროექტის სტრუქტურა ორგანიზებულია იტერაციულად, სადაც თითოეული მოდელი/არქიტექტურა განცალკევებულ ნოუთბუქშია წარმოდგენილი:

* `model1.ipynb` - საბაზისო არქიტექტურა (Baseline CNN) და საწყისი ექსპერიმენტები.
* `model2.ipynb` - გაღრმავებული CNN არქიტექტურა (Deeper CNN).
* `model3.ipynb` - Dropout რეგულარიზაციისა და Batch Size-ის ოპტიმიზაცია.
* `model4.ipynb` - მონაცემთა აუგმენტაცია (Data Augmentation) და Early Stopping.
* `model5.ipynb` - ResNet18 არქიტექტურა, აუგმენტაცია და Dropout ტუნინგი.
* `README.md` - პროექტის დოკუმენტაცია და ექსპერიმენტების ანალიზი.

## 🛠 სანითი ჩეკი (Sanity Check) & მოდელის შემოწმება

სანამ სრულყოფილ წვრთნას დავიწყებდით, განხორციელდა **Sanity Check** მილსადენის (Pipeline) და ფოროვარდ/ბექუორდ (Forward/Backward) პროცესების სისწორის შესამოწმებლად:
1. **საწყისი Loss-ის შემოწმება:** შემთხვევითი ინიციალიზაციისას 7 კლასისთვის თეორიული Loss უნდა იყოს $-ln(1/7) \approx 1.9459$. საწყისი Loss ამ მაჩვენებელთან ძალიან ახლოს აღმოჩნდა, რაც მიუთითებს კლასიფიკატორის სწორ საწყის მდგომარეობაზე.
2. **მცირე მონაცემებზე გადავსება (Overfitting on a small subset):** მოდელი გაიტესტა მხოლოდ 20 სურათისგან შემდგარ მცირე დატასეტზე 300 იტერაციით. მოდელმა შეძლო მონაცემების სრულად დამახსოვრება და Loss-ის ნულამდე დაყვანა, რაც ადასტურებს, რომ გრადიენტების გამოთვლა (Backward) და პარამეტრების განახლება გამართულად მუშაობს.

## 📈 იტერაციული განვითარება და ექსპერიმენტები

ლექტორის მოთხოვნის შესაბამისად, ჩვენ დავიწყეთ მარტივი არქიტექტურით და ეტაპობრივად დავამატეთ სხვადასხვა კომპონენტები:

### 🔬 ექსპერიმენტი 1: საბაზისო მოდელი (Baseline CNN)
* **ფაილი:** `model1.ipynb`
* **მიღებული გადაწყვეტილება:** დავიწყეთ მარტივი არქიტექტურით, რათა შეგვექმნა საწყისი წერტილი (Baseline). მოდელი შედგება 3 საკონვოლუციო (Conv2d) ფენისგან, Batch Normalization-ით, MaxPool2d-ით და ბოლოს Fully Connected (FC) ფენისგან.
* **ჰიპერპარამეტრები:** Learning Rate = `5e-4`, Batch Size = `64`, Epochs = `10`.
* **შედეგები (W&B Run: `model1_lr5e-4_bs64`):**
  * Train Accuracy: ~63.63%
  * Validation Accuracy: ~50.11%
  * Test Accuracy: **52.91%**
* **ანალიზი:** მოდელს აქვს გამოკვეთილი Overfitting-ის პრობლემა (სავარჯიშო სიზუსტე ბევრად აღემატება ვალიდაციისას). საჭიროა მოდელის ტევადობის გაზრდა ან რეგულარიზაციის დამატება.

### 🔬 ექსპერიმენტი 2: არქიტექტურის გაღრმავება (Deeper CNN)
* **ფაილი:** `model2.ipynb`
* **მიღებული გადაწყვეტილება:** Overfitting-ის მიუხედავად, მოდელს უჭირდა რთული პატერნების სწავლა, ამიტომ გადაწყდა ქსელის გაღრმავება და დამატებითი საკონვოლუციო ბლოკების დამატება არხების (Channels) რაოდენობის გაზრდით (მაგ. 64 -> 128 -> 256).
* **ჰიპერპარამეტრები:** Learning Rate = `5e-4`, Batch Size = `64`, Epochs = `10`.
* **შედეგები (W&B Run: `model2_deepercnn_lr5e-4_bs64_epochs10`):**
  * Train Accuracy: ~78.70%
  * Validation Accuracy: ~57.27%
  * Test Accuracy: **58.83%**
* **ანალიზი:** ქსელის გაღრმავებამ გააუმჯობესა ტესტის სიზუსტე (58.83%-მდე), თუმცა Overfitting კიდევ უფრო გაღრმავდა (Train Acc მიუახლოვდა 78.7%-ს). აუცილებელია ძლიერი რეგულარიზაციის მეთოდების შემოტანა.

### 🔬 ექსპერიმენტი 3: რეგულარიზაცია Dropout-ით
* **ფაილი:** `model3.ipynb`
* **მიღებული გადაწყვეტილება:** Overfitting-ის შესამცირებლად Fully Connected ფენებამდე დავამატეთ `Dropout(0.3)` ფენები, რაც შემთხვევითად თიშავს ნეირონების ნაწილს და აიძულებს ქსელს ისწავლოს უფრო მდგრადი ფიჩერები.
* **ჰიპერპარამეტრები:** Learning Rate = `5e-4`, Batch Size = `64`.
* **შედეგები (W&B Run: `model3_dropout_lr5e-4_bs64`):**
  * Train Accuracy: ~56.51%
  * Validation Accuracy: ~55.32%
  * Test Accuracy: **55.37%**
* **ანალიზი:** Dropout-მა წარმატებით შეამცირა სხვაობა Train და Validation სიზუსტეებს შორის (Overfitting შემცირდა), თუმცა მოდელს მეტი ეპოქა სჭირდება სრული კონვერგენციისთვის.

### 🔬 ექსპერიმენტი 4: მონაცემთა აუგმენტაცია (Data Augmentation) და ადრეული გაჩერება (Early Stopping)
* **ფაილი:** `model4.ipynb`
* **მიღებული გადაწყვეტილება:** მოდელის განზოგადების (Generalization) უნარის გასაზრდელად შემოვიღეთ მონაცემთა აუგმენტაცია (`RandomHorizontalFlip`, `RandomRotation`). ასევე დაემატა **Early Stopping**, რათა ავტომატურად შეწყდეს წვრთნა, თუ ვალიდაციის Loss აღარ უმჯობესდება.
* **ჰიპერპარამეტრები:** ეპოქების ლიმიტი გაიზარდა 25-მდე, რადგან აუგმენტაციის დროს სწავლას მეტი დრო სჭირდება.
* **შედეგები (W&B Run: `model4_augmented_earlystop`):**
  * Train Accuracy: ~52.56%
  * Validation Accuracy: ~56.82%
  * Test Accuracy: **58.20%**
* **ანალიზი:** აუგმენტაციამ მნიშვნელოვნად დაასტაბილურა მოდელი. მიუხედავად იმისა, რომ სავარჯიშო სიზუსტე დაიკლო (რადგან მონაცემები გართულდა), ტესტზე მივიღეთ სტაბილური 58.20%.

### 🔬 ექსპერიმენტი 5: ResNet18-ის გამოყენება და Dropout-ის ოპტიმიზაცია
* **ფაილი:** `model5.ipynb`
* **მიღებული გადაწყვეტილება:** კლასიკური CNN-ების შემდეგ გადავწყვიტეთ უფრო თანამედროვე და მძლავრი არქიტექტურის - **ResNet18**-ის გამოყენება (Residual Connections-ით), რომელიც სპეციალურად მოვარგეთ 1-არხიან (Grayscale) სურათებს და 7 გამომავალ კლასს, აუგმენტაციასთან და Dropout რეგულარიზაციასთან კომბინაციაში.
* **ჰიპერპარამეტრები:** Epochs = `14`, Dropout = `0.3`.
* **შედეგები (W&B Run: `model_resnet18_augmented_dropout0.3`):**
  * Train Accuracy: ~66.60%
  * Validation Accuracy: ~53.42%
  * Test Accuracy: **52.26%**
* **ანალიზი:** ResNet18-მა სწრაფად დაიწყო გადავსება (Overfitting), რაც მიუთითებს იმაზე, რომ FER2013 დატასეტის მცირე რეზოლუციისთვის (48x48) წინასწარ გათვლილი დიდი არქიტექტურები საჭიროებენ ჰიპერპარამეტრების კიდევ უფრო ფრთხილ გადარჩევას (მაგ. უფრო მაღალ Weight Decay-ს ან Learning Rate-ის შემცირებას Scheduler-ით).
