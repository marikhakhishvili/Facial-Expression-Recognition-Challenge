##  სანითი ჩეკი (Sanity Check) & მოდელის შემოწმება

სანამ ტრენინგს დავიწყებდი, გავაკეთე **Sanity Check** Pipeline და Forward/Backward პროცესების სისწორის შესამოწმებლად:
1. **საწყისი Loss-ის შემოწმება:** შემთხვევითი ინიციალიზაციისას 7 კლასისთვის თეორიული Loss უნდა იყოს $-ln(1/7) \approx 1.9459$. საწყისი Loss ამ მაჩვენებელთან ძალიან ახლოს იყო, რაც მიუთითებს კლასიფიკატორის სწორ საწყის მდგომარეობას.
2. **მცირე მონაცემებზე ოვერფიტი:** მოდელი გაიტესტა მხოლოდ 20 სურათისგან შემდგარ მცირე დატასეტზე 300 იტერაციით. მოდელმა შეძლო მონაცემების სრულად დამახსოვრება და Loss-ის ნულამდე დაყვანა, რაც ადასტურებს, რომ გრადიენტების გამოთვლა  და პარამეტრების განახლება გამართულად მუშაობს.
   
(თავიდან ყველა მოდელში ვაპირებდი გაკეთებას, მაგრამ მერე მივხვდი, რომ არ იყო საჭირო და მეორეს შემდეგ შევწყვიტე)

## მონაცემების დამუშავება
პიქსელების სტრინგი გადავიყვანე pandas array-ში და გავყავი ტრეინ სეტი სამად. ტრეინ, ვალიდაციის და ტესტ სეტებად (70/15/15). (თავიდან ტესტ სეტი რომ მჭირდებოდა ვერ მიხვდვი და მაგის გარეშე დავიწყე მოდელის ტრენინგი). იმისთვის, რომ თანაბრად გადანაწილებულიყო საბსეტებში ემოციები გამოვიყენე stratified splitting. და ბოლოს გავუკეთე ნორმალიზაცია.

##  ექსპერიმენტები

###  ექსპერიმენტი 1: Baseline CNN
* დავიწყე მარტივი არქიტექტურით. მოდელი შედგება 2 Conv2d ბლოკისგან, თითოეულთან ReLu და maxpooling გამოვიყენე, ბოლოს flattening. loss function crossEntropy ავიღე და ადამ ოპტიმაიზერი.
* თავიდან Learning Rate = `1e-3` მქონდა, მაგრამ რადგანაც ძალიან ოვერფიტში წავიდა გადავწყვიტე გამეზარდა და Learning Rate = `5e-4` გავხადე. ეპოქების რაოდენობა შევამცირე, რადგანაც ზედმეტად დიდხანს უნდებოდა (გვიან მივხვდი, რომ cpu-ს ვიყენებდი gpu-ს ნაცვლად). მიუხედავად ამ ცვლილებებისა მაინც ოვერფიტი იყო, ანუ მოდელის სიღრმის გაზრდა იყო საჭირო და შემდეგი მოდელის ტრენინგზე გადავედი.
* **შედეგები (wandb Run: `model1_lr5e-4_bs64`):**
  * Train Accuracy: ~63.63%
  * Validation Accuracy: ~50.11%
  * Test Accuracy: **52.91%**

###  ექსპერიმენტი 2: Deeper CNN

სიღრმე გავზარდე და batchnorm დავამატე, მაგრამ ამან ოვერფიტის პრობლემის მოგვარება ვერ შეძლო. გაიზარდა ვალიდაციის აქურესი, მაგრამ ოვერფიტი არ შემცირებულა, პირიქით უფრო გაიზარდა. ამიტომაც გადავწყვიტე დროფაუთის დამატება, რათა ოვერფიტი როგორმე შემემცირებინა.
* **ჰიპერპარამეტრები:** Learning Rate = `5e-4`, Batch Size = `64`, Epochs = `10`.
* **შედეგები (wandb Run: `model2_deepercnn_lr5e-4_bs64_epochs10`):**
  * Train Accuracy: ~78.70%
  * Validation Accuracy: ~57.27%
  * Test Accuracy: **58.83%**


###  ექსპერიმენტი 3: რეგულარიზაცია Dropout-ით

Overfitting-ის შესამცირებლად Fully Connected ფენებამდე დავამატე Dropout(0.3). რადგანაც იმ ჰიპერპარამეტრებზე, რომელბიც თავიდან ავირჩიე ისედაც კარგად იმუშავა სხვებზე აღარ მომისინჯავს, მაგრამ ალბათ აჯობებდა მეტ ეპოქაზე გამეშვა. (უბრალოდ ისევ cpu-ს ვიყენებდი gpu-ს ნაცვლად და 25 ეპოქაზე ერლი სტოპინგის გარეშე ძაან დიდ დროს ანდომებდა).
* **ჰიპერპარამეტრები:** Learning Rate = `5e-4`, Batch Size = `64`.
* **შედეგები (wandb Run: `model3_dropout_lr5e-4_bs64`):**
  * Train Accuracy: ~56.51%
  * Validation Accuracy: ~55.32%
  * Test Accuracy: **55.37%**


###  ექსპერიმენტი 4: Data Augmentation და Early Stopping
მოდელის გენერალიზაციის გასაზრდელად შემოვიღე მონაცემთა აუგმენტაცია (`RandomHorizontalFlip`, `RandomRotation`). ასევე დავამატე Early Stopping, რათა ავტომატურად შეწყდეს წვრთნა, თუ ვალიდაციის Loss აღარ უმჯობესდება.
* **ჰიპერპარამეტრები:** ეპოქების ლიმიტი გაიზარდა 25-მდე, რადგან აუგმენტაციის დროს სწავლას მეტი დრო სჭირდება.
* **შედეგები (wandb Run: `model4_augmented_earlystop`):**
  * Train Accuracy: ~52.56%
  * Validation Accuracy: ~56.82%
  * Test Accuracy: **58.20%**
* **ანალიზი:** აუგმენტაციამ მნიშვნელოვნად დაასტაბილურა მოდელი. მიუხედავად იმისა, რომ სავარჯიშო სიზუსტე დაიკლო (რადგან მონაცემები გართულდა), ტესტზე მივიღეთ სტაბილური 58.20%.

###  ექსპერიმენტი 5: ResNet18-ის გამოყენება და Dropout-ის ოპტიმიზაცია
იმის გამო, რომ CNN-ებმა საკმარისად კარგი შედეგი ვერ მომცეს, გადავწყვიტე resnet გამომეყენებინა რომელიც სპეციალურად მოვარგე Grayscale სურათებს და 7 გამომავალ კლასს. თავიდან რეზნეტი გავუშვი დროპაუტის და აუგმენტაციის გარეშე და 8 ეპოქაში დაიზეპირა დატასეტი. ამის შემდეგ დავამატე "augmentation": "RandomHorizontalFlip, RandomRotation", "dropout": 0.5. ამანაც 6 ეპოქაში ისწავლა ყველაფერი და ვალიდაცია აღარ იზრდებოდა. შემდეგ ვცადე weight_decay-ს გაზრდა და დროპაუტის შემცირება, მაინტერესებდა რამეს შეცვლიდა თუ არა დროპაუტის შემცირება და დიდად არაფერი შეიცვალა.
* **ჰიპერპარამეტრები:** Epochs = `14`, Dropout = `0.3`.
* **შედეგები (W&B Run: `model_resnet18_augmented_dropout0.3`):**
  * Train Accuracy: ~66.60%
  * Validation Accuracy: ~53.42%
  * Test Accuracy: **52.26%**
* **ანალიზი:** ResNet-მა სწრაფად დაიწყო Overfitting-ს მიუხედავად იმისა რა პარამეტრებს ვიყენებდით. რაც მიუთითებს იმაზე, რომ დატასეტის მცირე რეზოლუციისთვის (48x48) რთულია დიდი არქიტექტურების გამოყენება. უფრო დეტალური ჰიპერპარამეტრების გადარჩევას საჭიროებს ალბათ. და ასევე ფოტოების რესაიზინგიც კარგი იქნებოდა.
